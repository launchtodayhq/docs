## Prerequisites

If you are new to the codebase, read [Start Here](/getting-started/start-here)
first. Then complete the [Prerequisites](/getting-started/prerequisites).

## Steps

## 1. Fork & Install

```bash
# Fork the repository on GitHub, then clone your fork
git clone https://github.com/launchtodayhq/launch.git
cd launch

# Install all dependencies
pnpm install
```

## 2. Set Up Database

<Tabs>
  <Tab title="Docker (Recommended)">
    ```bash
    cd apps/api
    docker-compose up -d
    ```
  </Tab>
  <Tab title="Local PostgreSQL">
    ```bash
    # macOS
    brew install postgresql
    brew services start postgresql

    # Create database
    createdb launch
    ```
  </Tab>
</Tabs>

## 3. Configure Environment

```bash
# Create your API env file
cd apps/api
touch .env
```

Edit `.env` with your values (see full list in
[Backend Environment Variables](/backend/environment-variables)):

```bash
# Database
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/launch"

# Auth (generate a random string)
BETTER_AUTH_SECRET="your-super-secret-key-at-least-32-chars"
BETTER_AUTH_URL="http://localhost:3001"

# CORS
TRUSTED_ORIGINS="http://localhost:3000,http://localhost:19006,launch://"
```

## 4. Run Migrations

```bash
cd apps/api
pnpm prisma generate
pnpm prisma migrate dev
```

## 5. Start the Backend

```bash
cd apps/api
pnpm dev
```

You should see:

```
ðŸš€ Server running on http://localhost:3001
```

<Note>
  The mobile app needs the API and database running to work end-to-end. If the
  backend is not running, sign-in and data fetches will fail.
</Note>

## 6. Configure the Mobile App

Follow [Run the Mobile App](/getting-started/run-mobile) to set
`EXPO_PUBLIC_API_URL` and start Expo.

## 7. Start the Mobile App

Open the app on a device or simulator using Expo.

## 8. Test It Works

1. Open the app
2. Tap "Sign In"
3. Create an account with email
4. You should see the home screen!

<Check>**Congratulations!** You have Launch running locally.</Check>

- **Backend not running**: start `pnpm dev` in `apps/api`
- **Mobile canâ€™t reach API**: check `EXPO_PUBLIC_API_URL`
- **Database errors**: verify `DATABASE_URL` and run migrations
