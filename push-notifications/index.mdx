## Quick Start

### 1. Get Push Token

```typescript
import * as Notifications from "expo-notifications";

async function registerForPushNotifications() {
  const { status } = await Notifications.requestPermissionsAsync();
  if (status !== "granted") return;

  const token = await Notifications.getExpoPushTokenAsync();
  console.log("Push token:", token.data);

  // Send token to your backend
  await api.user.savePushToken({ token: token.data });
}
```

### 2. Send from Backend

```typescript
// apps/api/src/lib/push.ts
import { Expo } from "expo-server-sdk";

const expo = new Expo();

async function sendPushNotification(pushToken: string, message: string) {
  await expo.sendPushNotificationsAsync([
    {
      to: pushToken,
      title: "New Message",
      body: message,
      data: { screen: "messages" },
    },
  ]);
}
```

### 3. Handle in App

```typescript
Notifications.addNotificationResponseReceivedListener((response) => {
  const data = response.notification.request.content.data;
  if (data.screen) {
    router.push(data.screen);
  }
});
```

## Platform Requirements

| Platform | Requirement                     |
| -------- | ------------------------------- |
| iOS      | APNs key (from Apple Developer) |
| Android  | Firebase project (for FCM)      |

<Tip>
  For development, Expo Push works out of the box with Expo Go. Production
  requires APNs/FCM configuration.
</Tip>
